# PLAN

## 目的
- 男キャラ股間位置と女キャラ股間位置の距離を使い、腹ボコ値を自動制御する。
- 体位ごとに「最小へ落とし切る距離割合（%）」のみを主設定として保存・再利用する。

## 要件整理
- 距離 `d` は「男股間ターゲット」と「女股間ターゲット」のユークリッド距離を使う。
- 体位キーは `postureId + postureMode + motionStrength`（強弱分離）で管理する。
- 制御ルール:
  - 最短距離で最大腹ボコ値
  - 最短から `CutPercent` の位置で最小値へ到達
  - 以後、最長距離まで最小値を維持

## プロファイル構造（体位単位）
- `Enabled`
- `CutPercent` (0..1)
- `MinInflationSize`
- `MaxInflationSize`
- `DistanceMin`（学習値）
- `DistanceMax`（学習値）
- `LearnedCycles`（学習に使った往復数）
- `IsLearned`（学習済みフラグ）

## 実行フロー
1. 体位キーが未登録または未学習なら、学習モードで開始する。
2. 学習モード中は数ターン分（設定可能）距離を観測し、`DistanceMin/DistanceMax` を確定する。
3. 学習完了後、同キーのプロファイルに自動保存する。
4. 同セッション中も即反映し、次回以降は保存済みプロファイルを自動ロードして反映する。

## 制御計算
- `d_cut = DistanceMin + CutPercent * (DistanceMax - DistanceMin)`
- 腹ボコ値:
  - `d <= DistanceMin` -> `MaxInflationSize`
  - `DistanceMin < d < d_cut` -> `Max -> Min` を補間（既存イージング使用）
  - `d >= d_cut` -> `MinInflationSize`

## 安定化
- 距離ノイズ抑制として短い平滑化を入れる（移動平均 or 指数移動平均）。
- `DistanceMax == DistanceMin` 近傍は安全側で `MinInflationSize` へフォールバックする。

## UI方針
- 体位キーごとに `CutPercent` を編集可能にする。
- 学習状態（未学習/学習中/学習済み）、学習ターン数、現在の `DistanceMin/Max` を表示する。
- 手動で再学習開始・学習値リセット可能にする。

## 保存/読込方針
- 学習完了時に自動保存する。
- 体位キー一致時は自動ロードする。
- 未設定キーは学習モードへ自動遷移する。

## 実装ステップ
1. 距離サンプリング層を追加（男/女股間ターゲット取得 + 距離算出）。
2. 体位キーごとの学習ステート機構を追加。
3. 学習完了時のプロファイル保存と次回自動ロードを接続。
4. 既存腹ボコ適用経路に「距離ベース値」を差し込む。
5. UIに `CutPercent` と学習状態表示/操作を追加。
6. ログ観測点を追加（学習開始/完了/ロード/適用値）。

## 受け入れ条件
- 初回未設定体位で、指定ターン観測後に自動学習完了し、プロファイルが保存される。
- 同体位再突入時、学習なしで保存済みプロファイルが自動適用される。
- `CutPercent` 変更が即反映され、補間カーブの挙動が破綻しない。
- 強/弱モーションでプロファイルが混線しない。
