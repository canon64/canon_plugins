SpeedTimeline 完全編集ガイド

MainGameSpeedLimitBreak / 「どのキーを残して、どれを消すか」を迷わないための版

0. 先に結論（ここだけ読めば最低OK）

  • 1cueに 必要な機能のキーだけ書く。
  • 使わないキーは消す。特に TaiiId/TaiiMode/CoordinateType を残しっぱなしにしない。
  • "CoordinateType": "" みたいな型違いはNG。
  • 編集反映は Ctrl+R か再起動。
読み込むファイルは F:/kks/BepInEx/config/com.kks.maingame.speedlimitbreak.cfg の [Video Timeline] 01 Cue File Path で決まる。

1. どのファイルを編集するか

実際に使われるJSON 01 Cue File Path が指しているファイル

いまの例 speedTimeline.json を指しているなら、 F:/kks/BepInEx/plugins/MainGameSpeedLimitBreak/speedTimeline.json

同フォルダ内に似た名前のファイルが複数あっても、読まれるのは1本だけ。

2. cueの最小構造

{
  "TimeSec": 35,
  "Enabled": true,
  "(必要なキーだけここに追加)": "..."
}

TimeSecだけでは無効。機能キー（Preset/Face/Taii/Coordinate/Clothes/Click）のどれかが必要。

3. 用途別テンプレ（コピペ用）

3-1. 速度プリセットだけ

{ "TimeSec": 10, "PresetName": "正常位 286-71.5" }

3-2. 強弱切替だけ（motionchange）

{ "TimeSec": 15, "ClickKind": "motionchange" }

意味動画時刻が15秒を跨いだ瞬間に、1フレームだけ motionchange クリック命令を送る。

効果本編UIの「強弱切替」を押したのと同じ扱い。

重要ClickKind は文字列の命令名。衣装部位の Kind とは別物。

3-3. 体位だけ

{
  "TimeSec": 20,
  "TaiiName": "正常位",
  "TaiiMode": 3
}

3-4. コーデだけ

{ "TimeSec": 25, "CoordinateType": 1 }

3-5. 部位着脱だけ

{
  "TimeSec": 30,
  "ClothesStates": [
    { "Kind": 0, "State": 1 },
    { "Kind": 1, "State": 3 }
  ]
}

意味ClothesStates は配列で、1行が「部位1つの着脱命令」。

例{ "Kind": 0, "State": 3 } は「上着を脱衣状態にする」。

注意衣装変更したくないcueでは ClothesStates キー自体を消す。

3-6. 表情だけ（FaceDB）

{
  "TimeSec": 35,
  "FaceDbEnabled": true,
  "FaceDbChara": "c13",
  "FaceDbVoiceKind": 1,
  "FaceDbAction": 0,
  "FaceDbRandom": true
}

3-7. 「Kind」は3種類ある（混同しない）

キー名値の型意味例
ClickKind string Hシーンへ送るクリック命令名 "motionchange"
ClothesStates[].Kind int 衣装部位番号 0=上着, 1=ボトム
FaceDbVoiceKind int 表情DBの voice_kind 条件 0/1/2

4. キー辞書（編集時の基準）

キー型意味使う時使わない時
TimeSecnumber発火秒必須なし
Enabledbool有効/無効通常true省略可
TriggerOncebool一回のみ必要時のみ省略可
PresetNamestring保存プリセット適用速度変更したい時削除
GaugeSpeed131..3速度値（編集用の正規キー）速度を直接指定したい時削除
GaugePos010..1互換キー（旧形式）旧JSONとの互換削除
ClickKindstringクリック命令motionchange 等削除
TaiiNamestring体位名検索体位切替削除
TaiiIdint体位ID指定ID固定時削除
TaiiModeintモード絞り体位誤ヒット防止削除
CoordinateType0..3コーデ切替コーデ切替時削除
ClothesStatesarray部位着脱脱衣制御時削除
FaceDbEnabledboolFaceDB有効化表情適用時削除

5. あなたのブロック（TimeSec=105）を正しく削る

悪い例: 使わないキーが大量に残っていて、体位/コーデ/Faceの誤発火原因になる。

良い例（体位だけ）:

{
  "TimeSec": 105,
  "Enabled": true,
  "TaiiName": "正常位",
  "TaiiMode": 3
}

良い例（体位+速度）:

{
  "TimeSec": 105,
  "PresetName": "正常位 286-71.5",
  "TaiiName": "正常位",
  "TaiiMode": 3
}

6. FaceDBの場所（体位DBも同じ）

  • DB: F:/kks/work/FaceExpression/hface_extract/hface_dict.db
  • 表情: face_dict（VIEW） / hface_files, faces
  • 体位: h_anim_poses

7. ログ診断（困ったらここ）

  • [video-cue] triggered ... actionsApplied=yes なら何かは実行済み。
  • taii skipped reason=taii-not-found ... は体位条件が合ってない。
  • coordinate applied ... type=0 が毎回出るなら不要な CoordinateType が残ってる。
  • face skipped reason=face-db-no-match は FaceDB条件が厳しすぎる。
