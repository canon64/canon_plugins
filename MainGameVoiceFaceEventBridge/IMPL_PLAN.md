## 相談メモ（優先選択の合意）

- 先に基本体位（正常位/後背位/騎乗位/背面騎乗位/側位/座位/立位）を確定する
- 次に、その基本体位カテゴリ内で「◯◯抽出ワード」と一致する語を照合する
- 一致した場合は、該当体位を優先選択する
- 一致しない場合は、既存の通常選択ロジックにフォールバックする
- 複数一致した場合は「一致語が長いもの」を優先し、同長なら既存 priority で決定する

## 相談メモ（単語分離 + スコア化）

- 方針:
- 複合語（例: `腕引っ張り後背位`）はそのまま一致させず、単語に分離して加点する
- 先に基本体位カテゴリを確定し、そのカテゴリ内の候補体位だけでスコア計算する
- 最終スコアが閾値以上の候補を採用し、同点時は既存 priority → ランダムで決定する

- スコアの目安:
- 固有性が高い語（例: `マングリ`, `お姫様`, `フェンス`）: +40
- 中程度の語（例: `開脚`, `ブリッジ`, `背面`, `対面`）: +25
- 汎用語（例: `手`, `足`, `腰`, `膝`, `抱え`, `開`, `つき`）: +10
- 句読点・助詞・語尾は無視（例: `で`, `に`, `を`, `する`, `して`）

- 正常位系（分離語）:
- `開脚正常位` => `開脚(+25)` / `開(+10)` + `脚(+10)`
- `マングリ正常位` => `マングリ(+40)`
- `卓球台正常位` => `台(+25)`
- `膝抱え正常位` => `膝(+10)` + `抱え(+10)`
- `大股開き正常位` => `股(+25)` + `開き(+25)`
- `腰抱え正常位` => `腰(+10)` + `抱え(+10)`
- `ブリッジ正常位` => `ブリッジ(+25)`
- `ビーチボール正常位` => `ビーチ(+25)` + `ボール(+10)`

- 後背位系（分離語）:
- `腕引っ張り後背位` => `腕(+10)` or `手(+10)` + `引っ張(+40)`
- `フェンス後背位` => `フェンス(+40)`
- `バナナボート後背位` => `バナナ(+25)` + `ボート(+10)`
- `ネット後背位` => `ネット(+25)`
- `伏せ後背位` => `伏せ(+25)`
- `浮き輪後背位` => `浮き輪(+25)` or `浮き(+10)` + `輪(+10)`
- `足抱え後背位` => `足(+10)` + `抱え(+10)`

- 騎乗位系（分離語）:
- `ソファ騎乗位` => `ソファ(+25)`
- `手つなぎ騎乗位` => `手(+10)` + `つなぎ(+25)`
- `乳首責め騎乗位` => `乳首(+40)` + `責め(+25)`
- `バナナボート騎乗位` => `バナナ(+25)` + `ボート(+10)`

- 背面騎乗位系（分離語）:
- `壁手つき背面騎乗位` => `壁(+40)` + `手つき(+25)` or `手(+10)` + `つき(+10)`

- 側位系（分離語）:
- `机側位` => `机(+25)`
- `背面側位` => `背面(+25)` or `後ろ(+25)`
- `お姫様抱っこ側位` => `お姫様(+40)` + `抱っこ(+25)`

- 座位系（分離語）:
- `床対面座位` => `床(+25)` + `対面(+25)`
- `足掛け対面座位` => `足掛け(+25)` or `足(+10)` + `掛け(+10)` + `対面(+25)`
- `床背面座位` => `床(+25)` + `背面(+25)`
- `正座背面座位` => `正座(+25)` + `背面(+25)`
- `膝立て背面座位` => `膝立て(+25)` or `膝(+10)` + `立て(+10)` + `背面(+25)`

- 立位系（分離語）:
- `卓球台立位` => `台(+25)`

- 初期閾値（案）:

## 奉仕系自動移行 実装計画（本格版 / JSON主導）

### 目的

- 「奉仕」「ご奉仕」などの意図語が返答文に含まれた場合、挿入系ではなく奉仕系へ高確率で遷移させる。
- 奉仕系の中でも、語彙に応じてフェラ系/手コキ系/パイズリ系/クンニ系/69系/足コキ系/顔面騎乗系/キス・愛撫系を判別する。
- まずはコード改修を行わず、`pose_houshi_classified.json` と `pose_score_rules.json` の調整のみで実現する。

### 非目的

- 今回は体位実行APIや `TryPickPoseFromText` の分岐順そのものは変更しない。
- modeIntの算出ロジック変更や、別プラグイン連携は対象外とする。

### 完了条件（Definition of Done）

- 「奉仕するね」「ご奉仕するよ」で `no pose keyword matched` が出ず、奉仕系体位が実行される。
- 「口で」「手で」「胸で」「舌で」「69」「足で」「顔に跨って」などで期待サブカテゴリに寄る。
- 否定文（`しない/じゃない/やめる/止める/やらない`）で誤遷移しない。
- 挿入系語（`挿れ/突い/ピストン`）と同時出現時、奉仕語が弱い場合は無理に奉仕へ寄せない。
- JSON更新のみで反映でき、ロールバックもJSON復元のみで完了する。

### 対象ファイル

- `F:\kks\BepInEx\plugins\canon_plugins\MainGameVoiceFaceEventBridge\pose_houshi_classified.json`
- `F:\kks\BepInEx\plugins\canon_plugins\MainGameVoiceFaceEventBridge\pose_score_rules.json`

### 現行ロジック前提（変更しない）

- カテゴリ決定優先順:
- 1) 明示カテゴリ語一致
- 2) `inferRules` によるカテゴリ推定
- 3) グローバルトークンスコア推定
- 4) カテゴリ内で `rules` による体位優先
- 5) フォールバックランダム

- 本計画は上記フローを利用し、`inferRules` と `rules` を強化して精度を上げる。

### JSON設計（奉仕系カテゴリ導入）

- `pose_houshi_classified.json`:
- `categories` に `奉仕系` を追加。
- `奉仕系` の候補は以下8カテゴリを統合して構成する。
- `フェラ系`, `手コキ系`, `パイズリ系`, `クンニ系`, `69系`, `足コキ系`, `顔面騎乗系`, `キス・愛撫系`
- 重複除去キーは `modeInt + nameAnimation`。
- 並び順は `nameAnimation` 昇順、同名は `modeInt` 昇順に固定（差分管理しやすくする）。

- `pose_score_rules.json`:
- `categoryEnabled` に `奉仕系: true` を追加。
- `inferRules` に奉仕入口ルールと奉仕サブカテゴリ誘導ルールを追加。
- `rules` に奉仕系用のスコアルールを追加（カテゴリ確定後の体位選択精度向上）。

### inferRules 設計（カテゴリ推定）

- 入口ルール（最優先）:
- id: `infer_service_entry`
- targetCategory: `奉仕系`
- requiredAny: `奉仕`, `ご奉仕`
- excludeAny: `しない`, `じゃない`, `やめる`, `止める`, `やらない`
- priority: 190

- サブカテゴリ誘導（奉仕語が明示されないケースも拾う）:

- `infer_service_cunni`: クンニ系 / `舌`, `クリ`, `蜜壺`, `舐め`
- `infer_service_69`: 69系 / `69`, `シックスナイン`, `同時`
- `infer_service_foot`: 足コキ系 / `足`, `脚`, `挟`, `擦`, `しご`
- `infer_service_facesit`: 顔面騎乗系 / `顔面`, `顔`, `跨`, `乗`, `face sit`
- `infer_service_kiss_petting`: キス・愛撫系 / `キス`, `愛撫`, `焦ら`, `いちゃ`

### rules 設計（カテゴリ内体位選択）

- スコア閾値は既存準拠:
- `adoptThreshold=35`, `forceThreshold=60` を維持。
